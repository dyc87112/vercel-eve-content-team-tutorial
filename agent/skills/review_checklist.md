# Review Checklist

Use this procedure when reviewing an article draft.

Check:

1. Audience fit: Is it useful for Java/Spring developers?
2. Source quality: Are official docs or primary sources present?
3. Engineering usefulness: Does the draft contain runnable or easily runnable implementation details?
4. Version safety: Are Spring, Java, and dependency versions explicit or clearly marked as requiring verification?
5. Claim safety: Are performance, release, security, and compatibility claims backed by sources?
6. Publishing readiness: Does it need code, screenshots, diagrams, or manual fact checks before publishing?

Return:

- readiness status;
- score;
- strengths;
- risks;
- required fixes.
