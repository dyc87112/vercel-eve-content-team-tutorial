# Topic Planning Procedure

Use this procedure when the user asks for topics, a publishing plan, a weekly content radar, or ideas for SpringForAll.

Do not rely on fixed demo topic lists. Build the recommendation from fresh source discovery whenever search, browsing, RSS, repository, or file tools are available.

## Source discovery

Start from primary and near-primary sources:

1. Spring official sources:
   - Spring blog;
   - Spring Boot reference documentation and release notes;
   - Spring AI reference documentation and release notes;
   - Spring Cloud release notes.
2. Java official sources:
   - OpenJDK JEPs;
   - Java release notes;
   - relevant JDK project pages.
3. Engineering and ecosystem signals:
   - GitHub releases, issues, discussions, and trending repositories around Spring, Java, AI, MCP, RAG, observability, and developer tooling;
   - official docs for relevant AI or infrastructure projects;
   - trusted technical blogs only when primary sources are unavailable.
4. Local account context:
   - existing articles;
   - backlog notes;
   - sample requests or prior drafts.

## Candidate generation

For each candidate, record:

- source title;
- source URL or local path;
- source type: official, primary, community, local, or secondary;
- observed date or freshness clue;
- why it matters to Java/Spring developers;
- what still needs verification.

Generate topics only after collecting enough source anchors. If live search is unavailable, say so and clearly mark recommendations as based on local context.

## Selection rules

Prefer topics that satisfy all of these:
   - useful for Java/Spring developers within one week;
   - can become a practical tutorial or engineering analysis;
   - has at least one official or primary source anchor;
   - avoids broad AI hype unless it connects to backend implementation.

Score each candidate from 1 to 100 using these weights:

- freshness: 20;
- SpringForAll audience fit: 25;
- source quality: 20;
- practical implementation potential: 20;
- differentiation from generic AI commentary: 10;
- fit with existing account content: 5.

## Output

Present the top options with:

- title;
- angle;
- target reader;
- source anchors;
- freshness and uncertainty;
- difficulty;
- score;
- why now;
- recommended next action.

Mark anything that still needs live or manual verification.
