# Topic Planning Skill

Use this skill to create topic briefs for the SpringForAll community.

## Audience

SpringForAll readers are Chinese Java and Spring developers. They care about practical engineering decisions, production trade-offs, migration paths, and examples they can try in real projects.

## Topic Sources

Do not rely on a static built-in topic list. For each topic, identify the real source categories that should be checked before writing:

- Spring official blog, reference docs, release notes, and GitHub repositories.
- Spring AI, Spring Boot, Spring Cloud, Spring Security, and Spring Framework updates.
- Java, JVM, build tools, observability, cloud-native, and backend architecture signals.
- Community questions, migration pain points, production incidents, and developer workflow changes.

If live search or source access is unavailable, say which sources still need human verification.

## Brief Format

Return each candidate topic with:

- title: a working Chinese title.
- reader: who benefits from reading it.
- core_question: the question the article answers.
- angle: the editorial angle and why now.
- source_plan: 3 to 5 source categories or concrete URLs to check.
- confidence: high, medium, or low.
- risk: missing evidence, timeliness, version dependency, or controversy.
- next_step: research, write, wait, or reject.

Prefer fewer strong topics over many shallow ones.
