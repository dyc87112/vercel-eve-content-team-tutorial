# Review Checklist Skill

Use this skill to review a SpringForAll draft before the user treats it as publish-ready.

## Review Dimensions

Check:

- audience fit: whether the article helps Java and Spring developers.
- technical correctness: whether claims need version, source, or reproduction proof.
- evidence: whether each important claim has a source or is marked as an assumption.
- structure: whether the opening, explanation, examples, risks, and closing are coherent.
- usefulness: whether the reader gets an actionable next step.
- safety: whether the draft avoids invented facts, unsupported official-sounding claims, and auto-publication language.

## Result Levels

Use one of:

- pass: only minor edits remain.
- revise: useful draft, but must fix issues before publishing.
- block: do not publish until key facts or structure are rebuilt.

## Review Output

Return:

- verdict: pass, revise, or block.
- must_fix: issues that block publication.
- should_fix: quality improvements.
- fact_checks: claims that need human verification.
- edit_notes: concise rewrite suggestions.
- approval_note: what the user should confirm before publishing.
